## 1. 产品概述

### 1.1 背景与目标

公司现有人事系统在候选人筛选、薪酬核算等环节存在重复录入、人工判断不一致、业务流转分散的问题。本产品旨在构建一个 AI 人事系统 MVP，作为内部数字化工具的候选方案。

**产品目标：**
- 招聘环节：通过 AI 辅助实现从简历解析到 Offer 发放的端到端闭环
- 薪酬环节：提供员工管理和固定薪酬录入的最小可用能力
- AI 价值：AI 作为全流程助手，而非简单的聊天窗口

### 1.2 目标用户

| 用户角色 | 使用场景 | 核心诉求 |
|----------|----------|----------|
| 普通 HR | 日常招聘操作：录入候选人、推进流程、安排面试 | 提高筛选效率，减少重复录入 |
| HR 主管 | 审批 Offer、查看薪酬数据、管理用户 | 全局把控，关键决策把关 |

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| JD (Job Description) | 岗位描述，包含职位名称、部门、要求、职责、薪资范围 |
| 候选人 | 通过简历投递或手动录入进入系统的应聘者 |
| AI 初筛 | AI 对简历进行结构化解析并与 JD 匹配评分的过程 |
| 流程阶段 | 候选人在招聘流程中的当前状态（如：简历筛选、面试、Offer 等） |
| AI 建议 | AI 生成的辅助决策内容（评分、面试问题、话术等），需人工确认后生效 |

## 3. 用户流程

### 3.1 核心流程：候选人招聘全流程

```
HR 登录系统
    │
    ▼
创建/选择 JD (F2)
    │
    ▼
上传简历 (F3.2) ──→ AI 自动解析简历 (F4.1)
    │                    │
    │                    ▼
    │              AI 匹配评分 (F4.2)
    │                    │
    ▼                    ▼
HR 查看候选人列表，按 AI 评分排序 (F3.3)
    │
    ├── 淘汰 (F3.6) ──→ AI 生成拒绝话术 (F4.4) ──→ 结束
    │
    ▼
推进到电话面试 (F3.5) ──→ AI 生成面试问题 (F4.3)
    │                              │
    ▼                              ▼
填写面试反馈 (F5.2)         HR 查看/编辑问题
    │
    ├── 不通过 ──→ 淘汰
    │
    ▼
推进到一面 → 二面 → 终面（重复面试流程）
    │
    ▼
推进到背调
    │
    ▼
发起 Offer (F6.1) ──→ AI 生成 Offer 建议 (F4.5)
    │
    ▼
HR 主管审批 (F6.2)
    │
    ├── 驳回 ──→ HR 修改后重新提交
    │
    ▼
Offer 发放 ──→ AI 生成沟通话术 (F4.4)
    │
    ▼
入职 (ONBOARDING) 或 拒绝 (REJECTED)
```

### 3.2 辅助流程：员工薪酬管理

```
HR 主管登录
    │
    ▼
新增员工 (F7.3) ──→ 填写姓名、工号、部门、职位、入职日期
    │
    ▼
录入薪酬 (F7.4) ──→ 选择员工、月份、填写固定工资
    │
    ▼
确认薪酬（草稿 → 已确认）
    │
    ▼
查看薪酬记录 (F7.5) ──→ 按员工/月份筛选
```

## 4. AI 功能详细设计

### 4.1 简历解析 (F4.1)

**输入：** 简历文件（PDF/Word）的文本内容
**输出：** 结构化 JSON

```json
{
  "name": "张三",
  "phone": "13800138000",
  "email": "zhangsan@example.com",
  "education": [
    { "school": "北京大学", "degree": "硕士", "major": "计算机科学", "graduationYear": 2022 }
  ],
  "workExperience": [
    { "company": "某科技公司", "position": "高级工程师", "startDate": "2022-07", "endDate": null, "description": "..." }
  ],
  "skills": ["Java", "Spring Boot", "MySQL", "Redis"],
  "projects": [
    { "name": "某项目", "role": "核心开发", "description": "...", "technologies": ["Java", "Spring"] }
  ]
}
```

**校验规则：**
- 姓名不能为空
- 联系方式至少有一个（手机或邮箱）
- 解析失败标记为"需人工处理"，不阻塞流程

### 4.2 候选人匹配评分 (F4.2)

**输入：** 简历结构化数据 + JD 要求
**输出：** 评分结果

```json
{
  "totalScore": 82,
  "dimensions": {
    "skillMatch": 85,
    "experienceMatch": 80,
    "educationMatch": 90,
    "stability": 75
  },
  "strengths": ["技能栈与岗位要求高度匹配", "名校背景"],
  "weaknesses": ["工作年限略低于要求", "缺少大型项目管理经验"],
  "recommendation": "建议进入面试环节"
}
```

**校验规则：**
- 分数 0-100，超出范围视为异常
- 必须包含至少 1 个优势和 1 个劣势
- 必须给出推荐结论

### 4.3 面试问题生成 (F4.3)

**输入：** 候选人简历 + JD + 面试轮次（一面/二面/终面）
**输出：** 结构化问题列表

```json
{
  "questions": [
    {
      "category": "technical",
      "question": "请描述 Spring Boot 自动配置的原理",
      "focus": "框架理解深度",
      "referenceAnswer": "..."
    },
    {
      "category": "behavioral",
      "question": "请描述一次你解决技术难题的经历",
      "focus": "问题解决能力",
      "referenceAnswer": "..."
    },
    {
      "category": "scenario",
      "question": "如果线上服务突然 OOM，你会如何排查？",
      "focus": "应急处理能力",
      "referenceAnswer": "..."
    }
  ]
}
```

**校验规则：**
- 至少 3 个问题
- 必须包含 question 和 focus 字段
- 终面问题应侧重综合素质考察

### 4.4 沟通话术生成 (F4.4)

**场景类型：** INVITE_INTERVIEW / REJECT / OFFER_COMMUNICATION

**输出：**
```json
{
  "script": "您好，张先生/女士，我是 XX 公司的 HR ...",
  "keyPoints": ["强调公司优势", "确认面试时间", "发送面试邀请函"]
}
```

### 4.5 Offer 建议 (F4.5)

**输出：**
```json
{
  "salarySuggestion": {
    "min": 18000,
    "recommended": 20000,
    "max": 22000
  },
  "negotiationStrategy": "候选人期望 22000，可从年终奖和成长空间进行谈判",
  "riskAlerts": ["候选人目前在职，需关注离职周期", "有竞品 offer 可能"]
}
```

## 7. 非功能性需求

| 维度 | 要求 |
|------|------|
| 响应时间 | 页面加载 < 2 秒，API 响应 < 500ms（不含 AI 调用） |
| AI 调用 | 超时 30 秒，失败重试 2 次 |
| 并发 | 支持 50 人同时在线 |
| 浏览器 | Chrome 90+、Edge 90+、Firefox 90+ |
| 安全 | 密码 BCrypt 加密，JWT 认证，SQL 注入防护，XSS 防护 |
| 数据 | 敏感数据脱敏，操作全量审计 |

## 8. 后续扩展方向（非 MVP）

以下功能在 MVP 中不实现，但给出设计方向，供后续迭代参考：

### 8.1 薪酬模块扩展

- **薪酬组成项**：增加绩效奖金、补贴、扣款项，引入薪酬规则引擎
- **绩效联动**：绩效结果自动影响浮动薪酬计算
- **薪酬批次计算**：按月批量生成薪酬，支持预览、校验、确认流程
- **工资条 PDF**：生成标准格式工资条，支持下载和打印
- **AI 异常检测**：自动识别薪酬异常（如跨月波动过大、重复发放）

### 8.2 招聘模块扩展

- **批量简历导入**：支持 ZIP 压缩包上传，并发解析
- **面试日历**：可视化排期，冲突检测
- **多候选人对比**：并排查看候选人简历和评分
- **数据看板**：招聘漏斗转化率、平均招聘周期、渠道效果分析
- **外部平台对接**：合规前提下对接 Boss 直聘、猎聘等平台 API

### 8.3 系统扩展

- **消息通知**：邮件/短信/企业微信通知
- **数据导出**：候选人列表、薪酬报表导出 Excel
- **移动端适配**：响应式优化或小程序
