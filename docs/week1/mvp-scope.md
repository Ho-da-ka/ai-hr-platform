# MVP 范围定义

> AI 人事系统 MVP — 第 1 周交付物
>
> 本文档明确定义 MVP 的范围边界、优先级排序和交付标准。

---

## 1. MVP 目标

**一句话定义：** 2 个月内，独立交付一个 AI 辅助的招聘管理系统，覆盖从简历上传到 Offer 发放的完整闭环，并提供员工薪酬管理的最小可用能力。

**核心验证假设：**
1. AI 能否有效提升简历筛选效率（解析准确率 > 80%，评分一致性达标）
2. 招聘全流程线上化是否能减少 HR 重复操作
3. AI 建议机制是否能辅助 HR 决策而非干扰

---

## 2. 范围分层

### P0 — 必须（无此不可交付）

| 模块 | 功能 | 验收标准 |
|------|------|----------|
| 认证 | 用户登录/登出 | JWT 认证，接口权限校验正常 |
| 权限 | 角色 (HR/HR主管) | 菜单、按钮、数据三级权限 |
| JD | 创建、编辑、列表、详情 | CRUD 完整，数据持久化 |
| 候选人 | 手动录入、简历上传 | 文件保存成功，候选人记录创建 |
| AI | 简历解析 | 输出结构化 JSON，字段完整 |
| AI | 匹配评分 | 输出分数 + 原因 + 优劣势 |
| 流程 | 候选人状态流转 | 10 个状态可正确流转 |
| 流程 | 淘汰 | 任意阶段可淘汰，记录原因 |
| 前端 | 所有 P0 功能页面 | 可操作，数据正确展示 |

### P1 — 应该有（MVP 完整性）

| 模块 | 功能 | 验收标准 |
|------|------|----------|
| AI | 面试问题生成 | 按轮次生成 5-10 个分类问题 |
| AI | 沟通话术生成 | 三种场景（邀约/拒绝/Offer）可生成 |
| AI | Offer 建议 | 输出薪酬建议 + 谈判策略 |
| 面试 | 面试记录管理 | 创建、查看面试反馈 |
| Offer | 发起 + 审批 | HR 提交，HR 主管审批 |
| 薪酬 | 员工 CRUD | 增删改查正常 |
| 薪酬 | 固定薪酬录入/查看 | 按月录入，支持草稿/确认 |
| 审计 | 操作日志自动记录 | 增删改操作留痕 |
| 用户 | 用户管理 | HR 主管可创建/禁用用户 |
| 数据 | 脱敏展示 | 手机号脱敏，薪酬权限控制 |

### P2 — 可以有（时间允许）

| 模块 | 功能 | 验收标准 |
|------|------|----------|
| 审计 | 日志查询页面 | 按时间/用户/类型筛选 |
| 仪表盘 | 招聘数据概览 | 各阶段候选人数量统计 |
| Demo | 初始化脚本 | schema.sql + data.sql 一键初始化 |

## 3. 明确不做的事项

| 事项 | 原因 | 替代方案 |
|------|------|----------|
| 批量简历导入 | 开发成本 > 收益 | 单文件上传 |
| 面试日历排期 | 非核心流程 | 线下沟通 |
| 消息通知 | 依赖外部服务 | 线下通知 |
| 数据导出 Excel | 非核心 | 页面查看 |
| 移动端 App | 超出 MVP 范围 | 响应式布局 |
| 多语言 | 内部工具仅中文 | — |
| 外部平台自动对接 | 合规风险 | 手动导入 + 来源标记 |
| AI 自动执行操作 | 安全风险 | AI 只建议，不执行 |
| 绩效联动薪酬 | 复杂度高 | 仅固定工资 |
| 电子签章 | 依赖第三方 | 记录状态即可 |

## 4. 技术范围

### 4.1 包含

- 后端：Spring Boot 3.2 + MyBatis-Plus + Spring Security + JWT
- 前端：React 18 + TypeScript + Ant Design 5 + Redux Toolkit + Vite
- 数据库：MySQL 8.0 + Redis 7.x
- AI：Claude API 集成 + Provider 接口抽象
- 部署：Docker Compose（开发） + Nginx + JAR（演示）

### 4.2 不包含

- 微服务拆分
- CI/CD 流水线
- 自动化测试全覆盖（核心逻辑有测试即可）
- 性能压测
- 监控告警（Prometheus/Grafana）

## 6. 时间规划

```
第 1 周  ┃ 需求分析、PRD、MVP 范围（当前阶段）
第 2 周  ┃ 架构设计、技术选型、数据库设计、页面草图
第 3 周  ┃ 后端基础框架 + 认证权限 + JD 管理 + 候选人管理
第 4 周  ┃ AI 模块（简历解析 + 匹配评分）+ 流程状态管理
第 5 周  ┃ AI 模块（面试问题 + 话术 + Offer 建议）+ 前端完善
第 6 周  ┃ 薪酬模块 + 审计日志 + Offer 审批 + 权限完善
第 7 周  ┃ 稳定性修复 + 异常处理 + Demo 数据 + 文档补全
第 8 周  ┃ 最终测试 + Demo 准备 + 技术答辩 + 复盘
```

## 7. 成功标准

**演示场景（必须跑通）：**

1. 以普通 HR 登录 → 创建 JD → 上传简历 → 查看 AI 解析结果和匹配评分 → 推进流程到面试
2. 以 HR 主管登录 → 审批 Offer → 查看薪酬数据 → 查看审计日志
3. 完整候选人全流程：简历筛选 → 电话面试 → 一面 → 二面 → 终面 → 背调 → Offer → 入职

**质量标准：**
- AI 简历解析输出结构化 JSON，字段完整率 > 90%
- AI 评分附带匹配原因和优劣势分析
- 所有 AI 建议需人工确认，不自动执行
- 敏感数据（手机号、薪酬）按角色脱敏
- 核心操作有审计记录
- 系统可在 10 分钟内部署启动
